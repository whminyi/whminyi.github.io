hexo + github 博客搭建过程

2017-11-13

###前言:
看到别人家的博客都搭建的那么漂亮，自己当然也是心血来潮，但是搭建起来才知道原来里面还是有这么多坑的，特此记录，也让后面的同学少栽坑里了

###Agend:

  • 本地搭建hexo静态博客
  • 博客绑定GitHub
  • 域名绑定
  • 博客内容创建
  • 主题美化

##1.如何搭建本地hexo静态博客

相关软件的安装

这里主要是node.js ，git 和hexo的安装

  • node.js官网下载 Node.js

  • git官网下载 Git下载

    都是比较简单的软件，下载后直接安装即可；

    当然照例检测是否安装成功，输入node -v 和npm -v,查看最新版本号，看下是否安装成功；

    由于node.js会自动安装并写入环境变量，因此基本都会成功；但是很不幸我就是那个不成功的人，不过只是环境变量没自动配置，不过手动配置npm的变量到nodeJS的目录下即可；

    其次就是重头戏 hexo 的安装了，比较推荐的方法是使用npm 或者cnpm来安装，命令如下：

    npm install hexo -g

    cnpm install hexo -g //-g 是指定为全局，全局安装之后，在任何地方都可以调用，如果不加入 -g 的话，包会被下载到在当前目录下，只可以被当前目录以内的程序访问

如果你能安装成功，那就恭喜你，网络特别棒；由于npm的服务器在地球的另一边米国。数据传输需要通过海底电缆，因此速度非常慢，像我这样本命年运气不好的在请求的时候直接就超时了，因此需要为此次安装指定一个远程仓库；

npm install --registry=https://registry.npm.taobao.org

或者

npm i --registry=https://registry.npm.taobao.org

当然如果你嫌麻烦，可以直接安装他的中国兄弟 cnpm:

npm install -g cnpm --registry=https://registry.npm.taobao.org

执行以下命令 npm -v 查看是否安装成功，至此就跳过了被墙的坑了；

搭建正式的运行环境

工具都安装好后，接下来就是实际操作建站了；

  • 1.新建一个空白文件夹，如HexoBlog
  • 2.在该文件夹内右键运行git，输入：hexo init(生成hexo模板)
  • 3.生成模板后运行 npm install（目前貌似不用）

  • 4.最后运行 hexo server

    运行程序，访问本地localhost:4000可以看到博客已经搭建成功；

    至此一个最简单的本地静态博客已经搭建完成，但是我们的目的肯定不是这么简单，当然是希望将博客内容异地托管然后外部也可以访问，那么久接着看

##2.绑定GitHub

###配置GitHub公钥

运行git bash，设置user.name 和 user.password配置信息：

git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

生成 SSH 秘钥文件：
ssh-keygen -t rsa -C “你的GitHub注册邮箱”

然后直接三个回车即可，默认不需要设置密码；

然后找到生成的.ssh的文件夹中的id_rsa.pub密钥，将内容全部复制，在GitHub中打开GitHub_Settings_keys 页面，新建new SSH Key，Title为标题，任意填即可，将刚刚复制的id_rsa.pub内容粘贴进去，最后点击Add SSH key。

检测GitHub公钥设置是否成功，输入 ssh git@github.com ：
成功的话内容显示是：hi rundouble! you are successfully…（抱歉贴不了图片）

这里之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

关联hexo与GitHub

####在GitHub上创建自己的仓库

创建账号什么的我就不说了，直接说创建步骤吧：
登录进去创建github仓库，创建仓库(new repository)，特别要注意的是，你的博客地址与Owner一栏显示的名字必须相挂钩。且必须是xxx.github.io形式。xxx是指你的owner一栏显示的文字。

这样个人站点配置OK，过几分钟输入个人站点配置ok，过个几分钟输入https://账户名称.github.io/, 也能看到个人博客的初始状态，只是暂时没有内容。

####仓库关联GitHub
在你本地blog目录下找到_config.yml 文件中的如下配置：
deploy:
type: git
repo: 这里填入你之前在GitHub上创建仓库的完整路径，记得加上 .git （例如：https://github.com/tengzhangchao/tengzhangchao.github.io.git）
branch: master

此处基本已经大工告成了，输入以下命令：
hexo clean //清除缓存，可以不用这个操作
hexo -g //生成本地文件
hexo -d //推送到GitHub发布

即可在
https://账户名称.github.io/
看到自己的热乎乎的博客了，是不是既激动又兴奋？什么你说没有，那么你可能遇到下面的坑了：

  • hexo空格的坑
    config文件中的类似于type 和 repo等后面配置值的时候，值的前后都是由空格的：
    title：博客博客 //错误
    title： 博客博客 //正确

    很多都是这个原因导致的，多注意检查

##3.内容创建及主题美化：

####内容创建：

在MyBlog目录下执行：hexo new “我的第一篇文章”，会在source->_posts文件夹内生成一个.md文件，接下来你可以参考markdown语法来生成自己的博客内容喽

####主题美化
可以参考 官网 提供的样式下载到本地，然后再_config.yml中设置下载theme即可(具体下载的主题的GitHub文档上有详细介绍我也急不多说了)

不得不说坑的确挺多，踩过才知道，还是那句话，纸上得来终觉浅 绝知此事要躬行，祝大家都有个自己的帅气的博客记录自己成长的点点滴滴；
